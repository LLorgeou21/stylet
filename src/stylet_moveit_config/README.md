# stylet_moveit_config

MoveIt 2 configuration for the "arm" planning group (6-DOF UR5e; the needle-insertion actuator is intentionally excluded - see below).

## Overview

Generated by the MoveIt Setup Assistant, then hand-modified in several places to work with a real Gazebo simulation rather than mock hardware (see the top-level `ARCHITECTURE.md` for the full story - `gz_ros2_control` integration, clock synchronization, `robot_description` propagation to RViz).

| File | Purpose |
|---|---|
| `config/stylet.urdf.xacro` | Combines `stylet_description`'s robot model with the `<ros2_control>` hardware-interface tags - a single `/robot_description` shared by Gazebo and MoveIt. |
| `config/stylet.ros2_control.xacro` | `ros2_control` system definition: `gz_ros2_control/GazeboSimSystem` hardware plugin, joint interfaces for the arm and for the 3-stage needle actuator. |
| `config/stylet.srdf` | Planning group "arm" (`base_link` → `tool0` - **not** `needle_tcp`: the needle joint is prismatic, including it would add a redundant IK degree of freedom), named states (`home`, `up`, `replie`), self-collision exemptions. |
| `config/kinematics.yaml` | IK solver for "arm": KDL. |
| `config/moveit_controllers.yaml` | Maps the "arm" group to the `FollowJointTrajectory` action exposed by `arm_controller`. |
| `config/ros2_controllers.yaml` | `ros2_control` controller definitions: `arm_controller` (6 joints) and `needle_insertion_controller` (3 joints), both `JointTrajectoryController`, `use_sim_time: true` everywhere (required for clock consistency with Gazebo). |
| `config/joint_limits.yaml` | Velocity/acceleration scaling for trajectory execution. |

## Launch files

| File | Purpose |
|---|---|
| `launch/move_group.launch.py` | Starts `move_group` with the full MoveIt config dict (`use_sim_time=True`). |
| `launch/moveit_rviz.launch.py` | Starts RViz with the MoveIt plugin and the `stylet_ui` panel/marker pre-loaded (`config/moveit.rviz`). |
| `launch/demo.launch.py` | Stock MoveIt Setup Assistant demo (mock hardware, no Gazebo). **Do not run alongside `stylet_simulation`** - both would publish `/robot_description` independently and conflict. Useful only for a quick MoveIt-only sanity check. |

## Usage

Not a standalone package - requires `stylet_simulation`'s `sim.launch.py` to already be running (for `/robot_description`, controllers, and `/clock`). See `stylet_bringup` for the full startup sequence and timing.

## Dependencies

`moveit_ros_move_group`, `moveit_kinematics`, `moveit_planners`, `moveit_simple_controller_manager`, `moveit_configs_utils`, `moveit_ros_visualization`, `moveit_setup_assistant`, `controller_manager`, `robot_state_publisher`, `rviz2`, `stylet_description`, `tf2_ros`, `xacro`.
